What is Middleware in Express?
  - In Express.js, middleware is a function that runs between the request and the response.
  - It has access to:
  - req (request)
  - res (response)
  - next() (go to next step)



Simple idea
  - Think of middleware like a checkpoint system:
  - Request comes in → middleware checks/does something → then passes it forward → response is sent


Why middleware is important
  - Middleware is used for:

  - 1.Authentication
    - Check if user is logged in

  - 2. Logging
    - Record requests

  - 3. Blocking requests
    - Reject invalid users
  
  - 4.Parsing data
    - Convert JSON body into usable format



Why do we use Middleware in Express?
  - Middleware is used because it lets us control and process requests before they reach the final route.



Simple idea
  - Middleware = “checkpoint between request and response”
  - It helps you:
    - check things
    - modify data
    - block requests
    - run shared logic
  




Why it is IMPORTANT
  - Without middleware:
    - You would repeat same code in every route

  - With middleware:
    - You write logic once
    - It runs everywhere automatically
